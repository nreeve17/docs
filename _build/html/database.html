biota.database¶

-biota.database.get_sample_data

get_sample_data¶

Function for pulling down a query from the database, grabbing the feature table for the data, and optionally filtering out the bad features.

Parameters¶

query: dict
The pymongo query for the specific samples you would like.
targets: dict, optional
The collections you would like to use, if omitted they default to: samples: FieldData, data: DADA2Table, features: FeatureData
sample_method: {‘intersection’, ‘union’}
How to screen samples, using the intersection with available data or including all samples for the query, regardless.
use_bridge: bool, optional
Use if sample metadata is split between tables that are related by a bridge table. Default True. If False, then sample_params and sample_groupings are not used and samples are only queried from the targeted samples collection
filter_features: bool, optional
If True, do not return features that are unassigned at the Kingdom or Phylum level. Default True
filter_function: function
The function should take and return a dataframe of sample by features, uses biota.util.default_filter_function if one isn’t provided. Make None to disable filtering.
coll_1: str, optional
The first sample metadata collection from the database to query. Used to retrieve table_1. Default FieldData
coll_2: str, optional
The second sample metadata collection from the database to query. Default SequenceSampleUUID
bridge_coll: str, optional
The bridge collection in the database that will map the ids from table_1 to the ids in table_2. Default BridgeFieldSample
id_1: str, optional
The column in the bridge that correspond to the ids in table_1. Default FieldSampleUUID
id_2: str, optional
The column in the bridge that correspond to the ids in table_2. Default SequenceSampleUUID
group_col: str, optional
The column to group the merged meteadata tables by. Default is SequenceSampleUUID
db: pymongo.database.Database, optional
The authenticated pymongo database you wish to use. If None, the default biota database is used.
agg_groupings: dict, optional
A dictionary with mapping aggregate functions for columns after the merged table is grouped, if omitted default parameters are used. See biota.database.DEFAULT_GROUPINGS

Returns¶

samples: pd.DataFrame
The queried samples metadata in a dataframe.
data: pd.DataFrame
The sample by feature table for the above samples.
features: pd.DataFrame
The feature data that applies to the samples.

Examples¶

>>> query = {'Company': 'novo.oil.and.gas',
...          'FieldSampleType': 'wellcutting'}
>>> samples, data, features = get_sample_data(query)

>>> query = {'ExtractionPlate':
...          {'$in':
...          ['NECs.plate.175.20180301', 'necs.plate.217.20180531']}}
>>> params = {'coll_1': 'SequenceSampleData',
...           'coll_2': 'FieldData',
...           'id_1': 'SequenceSampleUUID',
...           'id_2': 'FieldSampleUUID'}
>>> db = get_database()
>>> samples, data, features = get_sample_data(query, db=db, **params)

df_from_record¶

This is a convenience function that converts a pymongo query result to a feature table.

Parameters¶

cursor: pymongo.cursor.Cursor
The result generated by calling collection.find() which will be made into the resulting DataFrame.
fillna: str or int, optional
The value to use to fill any missing NA/NaN values in the DataFrame

Returns¶

pd.DataFrame:
pandas DataFrame containing the pymong query results, with all NA/NaN values substitued if the fillna values was provided
